# 워크플로

## 개발 흐름
설계 → 검증 → 계획 → 구현 → 테스트 → 리뷰. 선행 산출물이 없으면 만들거나 사용자에게 확인한 뒤 다음 단계로 간다.

## 검증·독립 평가
- 검증 없이 완료 선언 금지 — 실행 가능한 체크(테스트/빌드/린트)의 증거로만 "완료"를 주장한다.
- 생성자가 자기 결과를 평가하지 않는다 — 독립 평가는 `reviewer` 에이전트나 상위 모델 advisor에 맡긴다(fresh context).

## advisor(상위 모델) 호출 시점
- 접근/해석을 확정하기 전
- 막혔을 때(같은 오류 반복·수렴 안 됨)
- 완료 선언 전(산출물을 먼저 durable하게 저장한 뒤)
- 불확실성이 높다고 판단될 때

## 컨텍스트 관리
- 압축보다 리셋. 세션 간 이어갈 진행 맥락·일회성 선호는 에이전트 메모리에 남긴다(별도 핸드오프 파일 금지).
- 규칙·결정·구조·반복 오류의 재발 방지책처럼 영속 사실은 설정 레포의 소스 OKF에 남긴다. 배포본 OKF는 클린 재배포 대상이고, AGENTS.md 변경은 관리 레포에서만 수행한다.
- OKF 학습 축적은 [OKF 학습 축적 루프](/learning/accumulation.md) 기준으로 처리한다.
- 작업 전 관련 메모리·히스토리(최근 커밋·이전 결정)를 확인하고, 실제 코드/파일로 현재 상태를 검증한 뒤 진행한다.

## 배치 실행
- 대규모 작업은 계획 수립 후 실행. 배치 단위로 진행하고 체크포인트에서 검증, 실패 시 즉시 중단.
