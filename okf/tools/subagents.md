# 서브에이전트 위임

## 직접 vs 위임
- 단순 파일/심볼 검색 → grep/glob/lsp 직접 사용(서브에이전트 불필요).
- 3회 이상 탐색이 필요한 넓은 조사 → 탐색 전담 서브에이전트(`explore`)에 위임.
- 독립적 조사 여러 건 → 서브에이전트 병렬 실행으로 메인 컨텍스트 보호.

## 병렬·격리
- 코드 변경이 포함된 병렬 작업 → 격리된 워크트리에서 수행(충돌 방지).
- 대규모 작업은 계획 수립 후 배치 실행, 체크포인트에서 검증.

## 에이전트 선택
- 역할별 에이전트와 사용 시점은 [에이전트 가이드](/agents/guide.md) 참조.
- 위임받은 에이전트도 작업 전 관련 OKF 개념을 확인하고 omp 기본 도구·스킬을 우선 사용한다.
