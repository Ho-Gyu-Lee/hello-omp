# 스킬 사용

omp는 스킬을 디스크에서 발견·로드한다(사용자/프로젝트 범위). 작업이 스킬과 맞으면 직접 처리하기 전에 스킬을 쓴다.

## 원칙
- 작업 유형이 스킬과 매칭되면 해당 스킬을 우선 사용한다(브레인스토밍·계획·리뷰·디버깅·성능·리서치·다이어그램 등).
- 스킬은 도메인 절차·체크리스트를 제공한다 — 같은 일을 임기응변으로 다시 만들지 않는다.
- 스킬 사용 여부는 결과로만 드러낸다 — 내부 절차/스킬명을 장황히 노출하지 않는다.

## 대표 사용 시점
- 구현 전 탐색적 설계 논의 → 브레인스토밍 스킬.
- 다단계 기능 → 계획 스킬로 작업 분해·의존성·검증 포인트.
- 코드 작성 완료/PR → 리뷰 스킬(품질·보안·유지보수성).
- 에러·스택트레이스·재현 안 되는 버그 → 체계적 디버깅 스킬(근본 원인·가설 테스트·최소 변경).
- 버전/날짜/수치 확인 → 웹 검색 스킬(검증 프로토콜).
- 관계/흐름/상태 전이 설명 → 다이어그램 스킬.

## 주의
- 커뮤니티 스킬/플러그인은 실행 전 의심 패턴(명령 인젝션 등) 점검 — [MCP 정책](/tools/mcp.md)의 공급망 보안 참조.
